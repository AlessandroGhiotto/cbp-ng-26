\section{Introduction}
\label{sec:introduction}

In modern high-performance superscalar architectures, exploiting Instruction-Level Parallelism (ILP) relies heavily on the efficiency of the instruction fetch unit. Because out-of-order execution architectures must speculatively fetch and execute instructions after conditional branches to sustain throughput, any branch misprediction incurs a significant recovery penalty. This penalty involves flushing the pipeline and redirecting execution, which severely degrades the Instructions Per Cycle (IPC) and increases energy overhead. Accurate and low-latency branch prediction is required for maximizing overall processor efficiency.

The objective of this project is to evaluate the trade-offs in branch prediction architectures under tight hardware complexity constraints. Using the Hardware Complexity (\textbf{HARCOM}) library, we perform a comparative study of classic branch predictors established in the literature, including global-history-based models (Gshare~\cite{McFarling1993}, GAG, GAP~\cite{Yeh1991TwolevelAT}), local-history-based models (PAG, PAP~\cite{Yeh1991TwolevelAT}) and more sophisticated predictors (BiMode~\cite{lee1997bimode}, TAGE~\cite{seznec2006case}, Perceptron~\cite{jimenez2001dynamic}), alongside their line-wide extensions for block-level prediction. We explore the design parameter space (varying table configurations, counter widths, and history register lengths) subject to a strict memory budget. The evaluation focuses on maximizing the Voltage-Frequency-Scaled Speedup (\textbf{VFS}) metric~\cite{michaudvfs}, which models the processor performance by balancing prediction accuracy (Mispredictions Per Kilo-Instruction, \textbf{MPKI}), hardware \textbf{latency}, and energy consumption (Energy Per Instruction, \textbf{EPI}). 
% Under the physical timing model in HARCOM, the energy, latency, and area costs of the predictor's logic and memory components are dynamically tracked.

% Specifically, the VFS score evaluates three primary design dimensions:
% \begin{enumerate}
%     \item \textbf{Prediction Accuracy:} Quantified as Mispredictions Per Kilo-Instructions (MPKI), which dictates the rate of wrong-path instruction execution.
%     \item \textbf{Processor Performance (IPC):} Strongly influenced by the latencies of the two prediction stages (Stage 1 and Stage 2) within the instruction fetch pipeline.
%     \item \textbf{Energy Efficiency:} Measured as dynamic and static Energy Per Instruction (EPI) in fJ, dictated by the capacitance, access frequencies, and leakage currents of the predictor's register arrays and SRAM tables.
% \end{enumerate}

% The VFS metric is computed using the CPU speedup ($S$) and the normalized EPI. The processor speedup is defined as:
% \begin{equation}
% \label{eq:speedup}
% S = \frac{\text{IPC}_{\text{cbp}}}{\text{IPC}_{\text{cbp}0}} \times \frac{1 + \text{WPI}_0}{1 + \text{WPI}}
% \end{equation}
% where $\text{WPI} = \text{IPC}_{\text{cbp}} \times \text{CPI}_{\text{cbp}}$ represents the wrong-path instructions per correct-path instruction, and the subscript $0$ indicates the reference core parameters. The normalized energy metric is formulated as:
% \begin{equation}
% \label{eq:epi}
% \begin{split}
% \text{EPI}_{\text{norm}} = {} & \left( \frac{\text{EPI}_{\text{cbp}}}{\text{EPI}_{\text{cbp}0}} \cdot r_{\text{energy}} + \Lambda \cdot S^\Gamma \right) \\
% & \times \left(1 + \frac{\text{WPI}}{2}\right)
% \end{split}
% \end{equation}

Building upon this comparative study, we propose a filterting scheme for the state-of-the-art \textbf{TAGE}~\cite{seznec2006case} (Tagged Geometric History Length) architecture. TAGE achieves excellent prediction accuracy by indexing multiple tagged predictor tables with increasing history lengths, but this comes at the cost of high dynamic Energy Per Instruction (EPI) due to parallel SRAM lookups. To address this energy overhead, we propose to identify highly biased or stable branches and predict them using simpler components, dynamically bypassing the TAGE tagged tables to minimize energy consumption and Stage 2 prediction latency.

\section{The HARCOM Framework}
\label{sec:harcom}

The simulation environment is built upon the Hardware Complexity (HARCOM) library. In HARCOM, predictors are implemented by inheriting from the \texttt{predictor} structure (a C++ \texttt{struct} that defines the abstract interface). This framework allows modeling physical hardware properties including area, power, and timing by tracking three fundamental data types: \texttt{val<N>}, \texttt{reg<N>} and \texttt{ram<T, N>}.

% \begin{itemize}
%     \item \textbf{\texttt{val<N>}}: Represents an $N$-bit value. Each \texttt{val} dynamically tracks its signal propagation time (latency) as it passes through combinational logic, alongside its dynamic and static energy costs.
%     \item \textbf{\texttt{reg<N>}}: Represents an $N$-bit hardware register, which persists state across clock cycles and is updated once per clock cycle.
%     \item \textbf{\texttt{ram<T, N>}}: Represents a single-port SRAM array of size $N$ with elements of type $T$. To model physical memory constraints, it permits at most one read or one write operation per cycle.
% \end{itemize}

The simulation interface separates the prediction and update processes. To minimize latency overhead on the processor's critical path, the interface supports \textbf{two-stage (two-level) prediction}. In Stage 1 (\texttt{predict1}), a fast, low-latency prediction is generated. In Stage 2 (\texttt{predict2}), a slower but more accurate prediction is computed, overriding the Stage 1 choice if they disagree. 

Furthermore, the interface enables \textbf{line prediction} (block-level prediction). The first instruction of a block is predicted via \texttt{predict1/2}, which usually performs a big table lookup on the SRAM and store it on the registers, subsequent instructions are predicted via \texttt{reuse\_predict1/2}, by reusing that values we have loaded previously. The updates are managed through the \texttt{update\_condbr} method (called after each conditional branch) and the \texttt{update\_cycle} method (called once per block to commit updates). 
%A block ends when we have a taken branch, a misprediction, or when the predictor logic decides that the block ends (call \texttt{reuse\_prediction(0)}). Figure~\ref{fig:harcom-interface} illustrates the flowchart of the interface calls within the simulator.

\section{Experiments}
\label{sec:exploration}

We evaluate our predictors under two memory budgets: \textbf{8 KB} and \textbf{16 KB}. To perform a comprehensive analysis across diverse application behaviors, we select a subset of 12 training traces from the 168 CBP-NG benchmark suite. These traces are chosen to represent extreme points and median statistics across key branch metrics, including branch density, conditional taken rate, backward branch rate, indirect branch rate, and unique PC counts, as summarized in Table~\ref{table:chosen-traces}. For each predictor family, we sweep design parameters (such as global history lengths, index widths, and tag sizes) to balance table capacity with indexing accuracy, seeking the configuration that maximizes the Voltage-Frequency-Scaled Speedup (\textbf{VFS}) metric within the budget constraints.

\subsection{Experimental Results}
Analyzing the simulation data (summarized for key configurations in Table~\ref{table:compact_summary}) reveals several insights into the trade-offs of branch predictor design:

\vspace{0.2cm}
\textbf{Trace Characteristics and Predictor Suitability.} \\
Simple global-history predictors (\texttt{gshareL}, \texttt{gapL}, and \texttt{gagL}) perform well on traces with low branch complexity. For instance, on \texttt{fp\_28} (which has a low branch density of 1.5\% and only 40 unique conditional PCs), \texttt{gshareL} achieves a high VFS score of 1.034, since aliasing in the prediction tables is negligible. Conversely, on complex workloads like \texttt{web\_13} (which contains 66,766 unique conditional PCs), the tagged geometric history length (\texttt{tageL}, \texttt{tage\_uL}) predictors outperform them significantly (VFS of 0.756 for \texttt{tageL} vs. 0.655 for \texttt{gshareL}). Tagged components successfully mitigate destructive aliasing across huge branch working sets. For traces with extremely high indirect rates (such as \texttt{tomcat-wrk2-panel.3289\_0} at 27.2\%), all conditional branch predictors suffer a general performance degradation due to the indirect branch recovery penalties.

\vspace{0.2cm}
\textbf{Block vs. Scalar Predictors.} \\
Block-level predictors (suffix \texttt{L}) fetch and predict multiple instructions in a single cycle (up to 16 instructions, as $LI=16$) by reusing the active SRAM row, leading to a massive IPC boost (ranging from 3.0 to 4.85) compared to scalar models (which are strictly limited to an IPC range of 0.49--0.98). More importantly, this throughput dominance translates directly into energy efficiency. For example, the block-level \texttt{gshareL} (16~KB) achieves an EPI of 145~fJ, which is nearly half the EPI of its scalar counterpart \texttt{gshare} (263~fJ), despite executing at a 7x higher IPC. This occurs because scalar predictors must perform a separate SRAM read access for every single instruction, whereas block predictors amortize a single wide SRAM row read over the entire fetch block, fetching subsequent predictions from low-power registers (using \texttt{reuse\_predict1/2}). This reduction in active SRAM accesses results in a 3.0x--5.6x speedup in VFS score for block-level models over their scalar counterparts.

\vspace{0.2cm}
\textbf{Predictor Groups and Behaviors.} \\
The global-indexed models (\texttt{gap}, \texttt{gag}, and \texttt{gshare}) behave very similarly, as they use variations of global history XORed or concatenated with the PC address. When comparing TAGE variants, \texttt{tageL} achieves a higher IPC (4.705 vs. 4.266) due to simpler update rules, whereas \texttt{tage\_uL} achieves a lower MPKI (4.19 vs. 5.42) by utilizing usefulness (\texttt{u}) bits to track and deallocate useless entries, highlighting the accuracy-throughput trade-off.

\vspace{0.2cm}
\textbf{The 16 KB vs. 8 KB Budget Trade-off.} \\
Doubling the budget from 8 KB to 16 KB reduces the branch misprediction rate, but it is not guaranteed to improve the VFS score. Higher capacity increases SRAM access latency and energy (e.g., \texttt{tageL} EPI increases from 939 fJ at 8 KB to 1145 fJ at 16 KB). As a result, the average VFS score for several predictors actually degrades when moving from 8 KB to 16 KB (e.g., \texttt{tageL} VFS drops from 0.710 to 0.699).

\subsection{Our Proposal}
Building upon the trade-off analysis of block-level TAGE predictors, we propose three distinct TAGE variants designed to optimize the base predictor for energy efficiency and prediction throughput:

\vspace{0.2cm}
\textbf{TAGE bimode.} \\
This variant replaces the simple bimodal base predictor ($t_0$) with a Bi-Mode predictor~\cite{lee1997bimode}, for increasing the accuracy of the base predictor. Under strict memory budgets (8~KB and 16~KB), the choice table, taken PHT, and not-taken PHT consume a significant fraction of the storage. This limits the capacity of the TAGE tagged tables, forcing them to use short tags and fewer rows, causing the average MPKI to degrade to 5.64 (at 16~KB) and 6.39 (at 8~KB) compared to 5.42 for \texttt{tageL}. It also incurs higher energy overhead.

\vspace{0.2cm}
\textbf{TAGE bias.} \\
To filter biased branches, we integrate a small, dedicated bias table (inspired by Bi-Mode++~\cite{kise2005bimode}) as a first-level filter. Strongly biased branches skip base-table updates and are predicted early. While this keeps the TAGE table size intact and achieves a highly accurate 5.25 MPKI and a lower EPI (932~fJ vs. 1145~fJ for \texttt{tageL}), it incurs a substantial IPC penalty (2.8389 vs. 4.7048). 
% Under HARCOM's physical timing model, the sequential dependency of the bias table lookup and prediction logic stretches the Stage 1 combinational critical path past the 300~ps clock period constraint, forcing Stage 1 to take 2 cycles instead of 1.

\vspace{0.2cm}
\textbf{TAGE sat.} \\
To overcome the latency bottleneck of the bias table while preserving energy savings, we propose a saturation-based bypass scheme. Instead of a separate bias table, we reuse the existing bimodal base table ($t_0$) counters: if a branch's bimodal counter is fully saturated (0 or max value), it is classified as strongly biased, and we dynamically skip the Stage 2 TAGE with the full table lookups. 

The experimental results show that \texttt{tage\_satL} is highly competitive:
\begin{itemize}
    \item \textbf{Throughput:} It achieves the highest average IPC among all evaluated predictors (4.8512 at 16~KB and 4.8579 at 8~KB). Bypassing the Stage 2 tagged RAM lookups for saturated branches eliminates their read latency.
    \item \textbf{Energy:} It reduces dynamic energy, achieving a 14.6\% reduction in EPI at 16~KB and a 21.3\% reduction at 8~KB compared to \texttt{tageL}.
    \item \textbf{VFS and MPKI:} Although bypassing TAGE increases the misprediction rate (MPKI rises to 7.39 at 16~KB), the combined throughput and energy benefits result in a high VFS score, demonstrating a highly efficient trade-off.
\end{itemize}

\section{Conclusion}
\label{sec:conclusion}

This project began with a broad study of classic branch prediction literature, identifying the trade-offs between prediction accuracy, latency, and hardware complexity. Our analysis highlighted that while the SotA TAGE predictor offers exceptional accuracy, its energy overhead (EPI) is substantial. To optimize this, we proposed simple bypassing schemes, such as a saturation-based filtering applied to the bimodal base predictor, which successfully trade a small amount of accuracy for EPI and latency savings. As future work, an idea could be to implement a collection of smaller, specialized predictors tailored to different branch behaviors. This would allow the system to select the most cost-effective predictor for each branch type, further reducing the energy and area overhead of relying entirely on a monolithic TAGE architecture.
